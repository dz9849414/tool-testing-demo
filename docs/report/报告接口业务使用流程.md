# 报告接口业务使用流程

## 📋 文档概述

本文档详细描述了报告与分析管理模块的业务使用流程，涵盖从模板创建到报告生成、统计分析的全流程操作指南。

---

## 🎯 核心业务流程

### 1. 模板管理流程

#### 1.1 创建模板（两种方式）

**方式一：通过API创建模板**
```bash
# 1. 创建基础模板
POST /api/report/templates
Content-Type: application/json

{
  "name": "效率分析报告模板",
  "description": "用于分析模板执行效率的报告模板",
  "templateType": "TEMPLATE_EFFICIENCY_STATISTICS",
  "templateStructure": "{\"template\": \"效率报告内容...\"}",
  "chapterStructure": "[{\"title\": \"概述\", \"type\": \"text\", \"dataSource\": \"overview\"}]",
  "content": "模板执行效率报告\\n总任务数：${totalJobs}",
  "styleConfig": "{\"theme\": \"light\", \"fontSize\": 14}",
  "isPublic": true
}

# 响应：返回模板ID
{
  "code": 200,
  "message": "操作成功",
  "data": 5
}
```

**方式二：通过XML导入模板**
```bash
# 1. 准备XML模板文件
efficiency_template.xml:
<?xml version="1.0" encoding="UTF-8"?>
<template>
  <name>效率分析报告模板</name>
  <description>模板执行效率分析报告</description>
  <templateType>TEMPLATE_EFFICIENCY_STATISTICS</templateType>
  <chapterStructure>[{"title":"概述","type":"text","dataSource":"overview"}]</chapterStructure>
  <content>模板执行效率报告\n总任务数：${totalJobs}</content>
  <styleConfig>{"theme":"light","fontSize":14}</styleConfig>
</template>

# 2. 上传XML文件
POST /api/report/templates/import
Content-Type: multipart/form-data

参数：
- file: @efficiency_template.xml
- newName: 我的自定义效率报告（可选）

# 响应：返回新创建的模板ID
{
  "code": 200,
  "message": "模板导入成功",
  "data": 6
}
```

#### 1.2 管理模板

```bash
# 查看模板列表
GET /api/report/templates?templateType=TEMPLATE_EFFICIENCY_STATISTICS

# 查看模板详情
GET /api/report/templates/5

# 更新模板
PUT /api/report/templates/5
Content-Type: application/json

{
  "name": "优化后的效率分析模板",
  "description": "优化后的效率分析报告模板"
}

# 导出模板为XML
GET /api/report/templates/5/export
# 响应：XML文件下载

# 删除模板（软删除）
DELETE /api/report/templates/5
```

### 3. 统计分析流程

#### 3.1 获取使用频率统计

```bash
# 获取定时任务使用统计（默认数据源）
GET /api/report/template-statistics/usage?timeRange=7DAYS

# 获取统一执行统计（手动+定时任务）
GET /api/report/template-statistics/usage?timeRange=7DAYS&dataSource=UNIFIED

# 获取批量任务统计
GET /api/report/template-statistics/usage?timeRange=30DAYS&dataSource=BATCH

# 获取自定义时间范围统计
GET /api/report/template-statistics/usage?startDate=2026-04-11&endDate=2026-04-21&dataSource=JOB_LOG

# 响应
{
  "templateStats": [
    {
      "templateId": 1,
      "templateName": "效率分析模板",
      "usageCount": 150,
      "successCount": 135,
      "failureCount": 15,
      "successRate": "90.00%",
      "avgDuration": "1200ms"
    }
  ]
}
```

**数据源选择说明**:
- **JOB_LOG**：定时任务数据源（template_job_log表）- 默认选项
- **UNIFIED**：手动+定时统一数据源（template_execute_log表）- 包含手动执行和定时任务
- **BATCH**：批量任务数据源（template_job_batch表）- 批量任务执行统计

#### 3.2 获取效率统计报告

```bash
# 获取定时任务效率报告（默认数据源）
GET /api/report/template-statistics/efficiency?startDate=2026-04-11&endDate=2026-04-21

# 获取统一执行效率报告（手动+定时任务）
GET /api/report/template-statistics/efficiency?startDate=2026-04-11&endDate=2026-04-21&dataSource=UNIFIED

# 获取批量任务效率报告
GET /api/report/template-statistics/efficiency?startDate=2026-04-11&endDate=2026-04-21&dataSource=BATCH

# 获取指定模板的效率报告
GET /api/report/template-statistics/efficiency?startDate=2026-04-11&endDate=2026-04-21&templateId=5&dataSource=UNIFIED

# 响应：完整的效率分析报告
{
  "id": 1776672532387,
  "name": "模板执行效率报告",
  "description": "基于模板5执行日志生成的效率分析报告",
  "content": "详细的效率分析内容...",
  "generateTime": "2026-04-21T10:30:00"
}
```

#### 3.3 生成图表数据

```bash
# 生成使用频率图表
GET /api/report/template-statistics/usage-chart?timeRange=7DAYS&chartType=BAR

# 生成效率趋势图表
GET /api/report/template-statistics/efficiency-chart?startDate=2026-04-11&endDate=2026-04-21&chartType=LINE
```

## 📊 典型业务场景

### 场景一：月度效率分析报告

**业务需求**：每月生成模板执行效率分析报告

**操作流程**：
1. **准备阶段**（每月1日）
   ```bash
   # 1. 检查是否有合适的模板
   GET /api/report/templates?templateType=TEMPLATE_EFFICIENCY_STATISTICS
   
   # 2. 如果没有，导入或创建模板
   POST /api/report/templates/import
   
   # 上传月度效率分析模板XML文件
   ```

2. **生成阶段**（每月2日）
   ```bash
   # 3. 生成月度报告
   POST /api/report/generate/template-efficiency
   Content-Type: application/json
   
   {
     "startTime": "2026-03-01",
     "endTime": "2026-03-31",
     "timeRange": "CUSTOM"
   }
   ```

3. **分发阶段**（每月3日）
   ```bash
   # 4. 导出报告
   GET /api/reports/{reportId}/export-pdf
   
   # 5. 发送邮件通知相关人员
   ```

### 场景二：实时监控仪表板

**业务需求**：实时监控模板使用情况

**操作流程**：
1. **设置数据源**
   ```bash
   # 1. 创建监控图表
   POST /api/report/charts
   Content-Type: application/json
   
   {
     "name": "实时使用频率监控",
     "chartType": "LINE",
     "dataSourceType": "TEMPLATE",
     "chartConfig": "实时监控配置"
   }
   ```

2. **前端轮询**
   ```javascript
   // 每30秒刷新一次数据
   setInterval(() => {
     fetch('/api/report/charts/{chartId}/data?startTime=last30min')
       .then(response => response.json())
       .then(data => updateDashboard(data));
   }, 30000);
   ```

### 场景三：故障排查分析

**业务需求**：分析模板执行失败原因

**操作流程**：
1. **获取失败记录**
   ```bash
   # 1. 查看失败时间线
   GET /api/reports/timeline?templateId=5&timeRange=TODAY
   
   # 2. 获取详细错误信息
   GET /api/report/template-statistics/usage?startDate=2026-04-21
   ```

2. **生成分析报告**
   ```bash
   # 3. 生成故障分析报告
   POST /api/report/generate
   Content-Type: application/json
   
   {
     "templateId": "5",
     "startTime": "2026-04-21T00:00:00",
     "endTime": "2026-04-21T23:59:59"
   }
   ```

## 🔄 自动化工作流

### 定时报告生成

**配置自动报告任务**：
```bash
# 1. 配置自动报告
POST /api/reports/auto-generate
Content-Type: application/json

{
  "reportType": "WEEKLY_EFFICIENCY",
  "dataSourceIds": "1,2,3",
  "generateFrequency": "WEEKLY",
  "nextGenerateTime": "2026-04-28T09:00:00"
}

# 2. 系统会自动在指定时间生成报告
# 3. 可以通过接口查看自动生成记录
GET /api/reports?generateType=AUTO
```

### 异常告警机制

**设置告警规则**：
```bash
# 1. 监控模板失败率
# 当失败率超过阈值时自动生成告警报告

Content-Type: application/json

{
  "name": "异常告警报告",
  "reportType": "ALERT",
  "content": "模板5失败率超过10%，请及时处理",
  "isScheduled": true,
  "alertCondition": "failureRate > 0.1"
}
```

## 📈 性能优化建议

### 1. 数据库优化
- 为常用查询字段建立索引（template_type, create_time, status）
- 定期清理历史报告数据
- 使用分页查询避免大数据量加载

### 2. 缓存策略
- 缓存模板结构数据
- 缓存常用的统计结果
- 使用Redis缓存图表数据

### 3. 异步处理
- 报告生成使用异步任务
- 大数据量导出使用后台任务
- 邮件通知使用消息队列

## 🔐 安全注意事项

### 1. 权限控制
- 敏感操作需要管理员权限
- 模板访问控制（公开/私有）
- 数据导出权限分离

### 2. 数据安全
- 参数校验防止SQL注入
- 文件上传类型限制
- XML解析安全防护

### 3. 审计日志
- 记录所有报告操作
- 跟踪模板修改历史
- 监控异常访问行为
