# 报告接口参数校验与错误处理指南

## 📋 文档概述

本文档详细描述了报告与分析管理模块的参数校验规则和错误处理机制，帮助开发人员避免常见的数据库字段验证错误，并提供清晰的错误提示信息。

## 🔧 参数校验规则

### 1. 通用校验规则

#### 1.1 必填字段校验
- **规则**：所有必填字段必须进行非空校验
- **错误提示**："字段名不能为空"
- **示例**：
```java
if (reportDTO.getName() == null || reportDTO.getName().trim().isEmpty()) {
    return Result.error("报告名称不能为空");
}
```

#### 1.2 JSON格式校验
- **规则**：所有JSON格式的字段必须进行格式验证
- **错误提示**："字段名必须是有效的JSON格式"
- **示例**：
```java
// 验证JSON格式
try {
    com.alibaba.fastjson2.JSON.parse(jsonString);
} catch (Exception e) {
    return Result.error("字段名JSON格式不正确");
}
```

#### 1.3 日期格式校验
- **规则**：日期字段必须符合yyyy-MM-dd格式
- **错误提示**："日期格式不正确，请使用yyyy-MM-dd格式"
- **示例**：
```java
try {
    java.time.LocalDate.parse(dateString);
} catch (Exception e) {
    return Result.error("日期格式不正确，请使用yyyy-MM-dd格式");
}
```

## 📊 各接口参数校验详情

### 2. ReportController - 报告管理接口

#### 2.1 创建报告 (`POST /api/reports`)

**必填字段校验**：
- `name`：报告名称（不能为空）
- `reportType`：报告类型（不能为空）

**JSON格式处理**：
- `dataSourceIds`：自动转换为JSON数组格式
  - 输入：`"1,2,3"` → 输出：`"[1,2,3]"`
  - 空值处理：设置为`"[]"`
- `chartIds`：自动转换为JSON数组格式
  - 输入：`"4,5,6"` → 输出：`"[4,5,6]"`
  - 空值处理：设置为`"[]"`

**错误提示示例**：
```json
{
    "code": 400,
    "message": "报告名称不能为空",
    "data": null
}
```

#### 2.2 自动生成报告 (`POST /api/reports/auto-generate`)

**必填字段校验**：
- `reportType`：报告类型（不能为空）
- `dataSourceIds`：数据源ID（不能为空）

**JSON格式处理**：
- `dataSourceIds`：自动转换为JSON数组格式
  - 输入：`"1,2,3"` → 输出：`"[1,2,3]"`
  - 格式验证：必须为有效JSON

**错误提示示例**：
```json
{
    "code": 400,
    "message": "数据源ID格式不正确，请使用逗号分隔的ID列表或JSON数组格式",
    "data": null
}
```

### 3. ReportTemplateController - 模板管理接口

#### 3.1 创建模板 (`POST /api/report/templates`)

**必填字段校验**：

**JSON格式校验**：

  - 空值处理：设置为`"{}"`
- `styleConfig`：必须是有效JSON对象
  - 空值处理：设置为`"{}"`

**错误提示示例**：
```json
{
    "code": 400,
    "message": "章节结构JSON格式不正确",
    "data": null
}
```

#### 3.2 XML模板导入 (`POST /api/report/templates/import`)

**文件校验**：
- 文件不能为空
- 文件必须是XML格式
- 文件大小限制检查

**错误提示示例**：
```json
{
    "code": 400,
    "message": "只支持XML格式的文件",
    "data": null
}
```

### 4. TemplateStatisticsController - 统计分析接口

#### 4.1 获取模板执行效率报告 (`GET /api/report/template-statistics/efficiency`)

**必填字段校验**：
- `startDate`：开始日期（不能为空）

**日期格式校验**：

## 🛡️ 数据库字段验证错误预防

### 6. 常见数据库错误及解决方案

#### 6.1 JSON字段验证错误
**错误信息**：
```
Data truncation: Invalid JSON text: "The document is empty." at position 0 in value for column 'report.data_source_ids'
```

**解决方案**：
```java
// 在Controller层进行预处理
if (dataSourceIds == null || dataSourceIds.trim().isEmpty()) {
    reportDTO.setDataSourceIds("[]");
} else {
    String processedIds = dataSourceIds.trim();
    if (!processedIds.startsWith("[") && !processedIds.startsWith("{")) {
        processedIds = "[" + processedIds + "]";
    }
    reportDTO.setDataSourceIds(processedIds);
}
```

#### 6.2 必填字段缺失错误
**错误信息**：
```
Column 'name' cannot be null
```

**解决方案**：
```java
// 在Controller层进行校验
if (reportDTO.getName() == null || reportDTO.getName().trim().isEmpty()) {
    return Result.error("报告名称不能为空");
}
```

#### 6.3 日期格式错误
**错误信息**：
```
Incorrect datetime value: 'invalid-date' for column 'create_time'
```

**解决方案**：
```java
// 在Controller层进行日期格式校验
try {
    java.time.LocalDate.parse(dateString);
} catch (Exception e) {
    return Result.error("日期格式不正确，请使用yyyy-MM-dd格式");
}
```

## 🔄 参数处理流程

### 7. 完整的参数处理流程

```java
public Result<Long> createReport(@RequestBody ReportDTO reportDTO) {
    try {
        // 1. 必填字段校验
        if (reportDTO.getName() == null || reportDTO.getName().trim().isEmpty()) {
            return Result.error("报告名称不能为空");
        }
        
        // 2. JSON格式预处理
        if (reportDTO.getDataSourceIds() != null && !reportDTO.getDataSourceIds().trim().isEmpty()) {
            String dataSourceIds = reportDTO.getDataSourceIds().trim();
            if (!dataSourceIds.startsWith("[") && !dataSourceIds.startsWith("{")) {
                reportDTO.setDataSourceIds("[" + dataSourceIds + "]");
            }
        } else {
            reportDTO.setDataSourceIds("[]");
        }
        
        // 3. JSON格式验证
        try {
            com.alibaba.fastjson2.JSON.parse(reportDTO.getDataSourceIds());
        } catch (Exception e) {
            return Result.error("数据源ID格式不正确");
        }
        
        // 4. 业务逻辑处理
        Long reportId = reportService.createReport(reportDTO);
        return Result.success(reportId, "报告创建成功");
        
    } catch (Exception e) {
        return Result.error("创建报告失败：" + e.getMessage());
    }
}
```

## 📝 错误码规范

### 8. 统一的错误码定义

| 错误码 | 错误类型 | 说明 |
|--------|----------|------|
| 400 | 参数错误 | 请求参数格式不正确或缺失 |
| 401 | 权限错误 | 用户未授权或权限不足 |
| 404 | 资源不存在 | 请求的资源不存在 |
| 500 | 服务器错误 | 服务器内部处理错误 |

    "code": 400,

## 💡 最佳实践建议

### 10. 前端参数处理建议

#### 10.1 数据源ID格式
- **推荐格式**：`"1,2,3"`（逗号分隔）
- **系统自动转换**：`"[1,2,3]"`（JSON数组）
- **避免直接发送**：复杂的JSON结构

#### 10.2 日期参数格式
- **推荐格式**：`"2026-04-21"`
- **避免格式**：`"2026/04/21"`、`"21-04-2026"`

#### 10.3 JSON配置参数
- **推荐格式**：完整的JSON对象
- **示例**：`{"theme": "light", "fontSize": 14}`
- **避免格式**：不完整的JSON片段

### 11. 测试用例建议

#### 11.1 边界值测试
- 空字符串：`""`
- 空格字符串：`"   "`
- null值：`null`
- 超长字符串：超过数据库字段限制

#### 11.2 格式错误测试
- 无效的JSON：`"invalid json"`
- 无效的日期：`"2026-02-30"`
- 特殊字符：`"<script>alert('xss')</script>"`
